<h3>Phase 2 : Stochastic Modelling (Handling Complex Navigation)</h3>

- Problem: Students navigate the website in an unpredictable manner, creating continuous loops that break standard visualization paths
- Solution: We implement a Markov-based sequence analysis

- Implementation : Instead of counting total clicks, we calculate the transition probabilities for every individual in the 2420 and 2510 cohorts. We will build a matrix showing the exact mathematical probability that a student moves from State A to State B.

- Robustness : Because our data includes sparse, rare events (like rage clicks), this mathematical matrix ensures our algorithms do not crash when a student performs a highly unusual navagational sequence.

<h3>Phase 3 : Explainable AI (linking behavior to course quality)</h3>

- For this phase, we need the students' grade data if we want to use it.
- Problem: Standard predictive algorithms operate as a "black box", telling us who will fail but offering no guidance on how to fix the course. 

- Solution : We will use the seqMAC framework, pairing our transition matrices with Stochastic Gradient Boosting and SHAP
