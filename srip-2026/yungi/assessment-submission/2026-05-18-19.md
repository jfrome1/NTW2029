# Sequence-Based Markov Machine Learning Classification

<h2>Research Objective and Problem Statement</h2>

<h4>Problem : The researchers identified a major flaw in how educational telemetry is analyzed</h4>

- Machine Learning is great at predicting "if" a student will fail , but it relies on static aggregated counts ("total logins") and acts as a "black box", offering no pedagogical explanation. 

- Markov models are great at explaining the temporal flow of learning (the "how"), but they completely break down when faced with "inter-sequence heterogeneity". So, if a student in the test data does something rare that was not in the training data, the Markov classifier crashes.

<h4>Objective : To create a hybrid model that can predict student success using the chronological flow of their clicks, while remaining mathematically explainable and resilient to highly varied, unpredictable student behavior.</h4>

<br>

<h2>Methodology: <b>seqMAC</b> Framework</h2>

1) Phase 1 : Dynamic Feature Generation Feature Generation (Markov Part) : Instead of counting "how many times" a student clicked, they use Markov Chain Models to calculate the <b>transition probabilities</b> between clicks for every individual student. For example, if a student never performs any transition, the probability would be wrriten as 0 in the matrix. This solves the heterogeneity crash problem.


2) Phase 2 : Prediction (Machine Learning) : they feed this matrix of transition probabilities into a supervise machine learning algorithm called the Stochastic Gradient Boosting algorithm to classify students into "High Performers" and "Low Performers" depending on their transitions within the website.


3) Phase 3 : Post-Hoc Interpretation : To avoid the "black box", they apply Explainable AI techniques such as the "Permutation Feature Importance and SHAP values), which tells them exactly "which specific behavioral transitions were most responsible for a student passing or failing.
